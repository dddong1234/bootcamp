# Day 3 — 흉부 X-ray 이상 탐지 모델 개선 과제

> 데이터: VinBigData Chest X-ray (512px 리사이즈 버전)
> 베이스라인: `Day3_baseline.ipynb` — YOLOv8n, imgsz=512, epoch=3, 단일 클래스 `abnormality`
> 제출물: 각 작업(Task)별 실험 리포트 + 최종 결과표 + 짧은 회고

---

## 시작하기 전에 (배경)

당신은 의료영상 스타트업의 ML 엔지니어입니다. 흉부 X-ray에서 이상 소견(abnormality)을 탐지하는 모델의 첫 베이스라인이 막 나왔습니다. 베이스라인은 **돌아가긴 하지만 성능이 낮습니다.** epoch 3, 가장 작은 모델, 단일 클래스로 "일단 파이프라인이 도는지"만 확인한 상태입니다.

팀 리드가 당신에게 개선을 맡겼습니다. 아래는 리드가 정리한 작업 티켓입니다. **각 티켓은 "무엇을 달성해야 하는가"만 적혀 있고, 구체적으로 어떤 하이퍼파라미터를 어떻게 바꿀지는 당신이 정합니다.**

### 작업 규칙

- 한 번에 하나의 변수만 바꿉니다. 두 개를 동시에 바꾸면 무엇이 효과를 냈는지 알 수 없습니다.
- 실험마다 `EXP_NAME`을 다르게 주어 `runs_cxr/` 결과가 덮이지 않게 합니다.
- 각 실험은 baseline과 **같은 조건에서 한 가지만** 다르게 해서 비교 가능해야 합니다.
- 먼저 baseline을 그대로 한 번 돌려 기준 숫자를 확보한 뒤 시작합니다.

---

## TICKET-001 — 작은 병변이 탐지되지 않는다

**우선순위**: High

**배경**
방사선사 검수 결과, 결절(nodule)이나 석회화(calcification)처럼 작은 소견이 모델에서 자주 누락된다는 피드백이 왔습니다. 현재 입력 해상도에서는 이런 작은 병변이 뭉개질 가능성이 있습니다.

**목표**
입력 해상도가 작은 병변 탐지에 미치는 영향을 검증하고, 개선 여부를 수치로 보인다.

**힌트로 주어진 제약**
- GPU 메모리 한계상 해상도를 올리면 배치 크기를 줄여야 할 수 있습니다.
- 해상도를 올리면 학습 시간이 늘어납니다. 시간도 함께 기록하세요.

**Acceptance Criteria**
- [ ] baseline 대비 recall 변화를 수치로 제시
- [ ] 학습 시간 증가분 기록
- [ ] 해상도를 올린 것이 비용 대비 가치가 있는지 한 문장으로 판단

**Deliverable**: 실험 리포트 1건 (아래 양식)

## TICKET-002 — mAP가 사업화 기준에 못 미친다

**우선순위**: High

**배경**
현재 mAP는 제품 데모에 쓰기엔 낮습니다. 가장 손쉬운 개선 레버 중 하나는 모델 용량을 키우는 것입니다. 다만 용량을 키우면 추론 속도와 학습 비용이 올라가므로 트레이드오프를 확인해야 합니다.

**목표**
모델 용량을 키웠을 때 성능이 오르는지, 그 대가로 학습/추론 비용이 얼마나 늘어나는지 측정한다.

**Acceptance Criteria**
- [ ] baseline 대비 mAP50, mAP50-95 변화 제시
- [ ] 학습 시간(또는 epoch당 시간) 변화 기록
- [ ] "성능 향상이 비용 증가를 정당화하는가"에 대한 판단

**Deliverable**: 실험 리포트 1건

---

## TICKET-003 — 더 학습시키면 더 좋아지는가

**우선순위**: Medium

**배경**
베이스라인은 epoch 3으로 끊겼습니다. 모델이 아직 덜 학습된 상태일 가능성이 높습니다. 다만 무작정 오래 돌리는 것은 비용 낭비이므로, **언제 멈춰야 하는지**를 데이터로 판단해야 합니다.

**목표**
학습을 더 진행했을 때 성능이 계속 오르는지, 어느 지점에서 정체되는지 파악한다.

**Acceptance Criteria**
- [ ] `results.png`의 loss 곡선이 아직 내려가는 중인지 / 평평해졌는지 판단
- [ ] 성능이 정체되기 시작하는 대략적 epoch 지점 제시
- [ ] "추가 학습이 가치 있는가"에 대한 판단

**Deliverable**: 실험 리포트 1건 (loss 곡선 캡처 포함)

## TICKET-004 — 모델이 "정상"으로 편향된다

**우선순위**: Medium

**배경**
데이터셋에는 'No finding'(정상) 이미지가 압도적으로 많습니다. 모델이 "아무 이상 없음"으로 답하기만 해도 손실이 낮아지는 구조라, 실제 병변을 놓치는 방향으로 편향될 수 있습니다.

**목표**
정상 이미지 비중이 모델 편향에 미치는 영향을 검증하고, 데이터 구성 변경이 recall에 주는 효과를 확인한다.

**힌트로 주어진 제약**
- 데이터 구성을 바꾸면 precision과 recall이 서로 다른 방향으로 움직일 수 있습니다. 둘 다 보세요.

**Acceptance Criteria**
- [ ] 데이터 구성 변경 전/후 precision, recall 비교
- [ ] 이 변경의 부작용(있다면) 서술
- [ ] 실제 제품이라면 이 구성을 채택할지 판단

**Deliverable**: 실험 리포트 1건

## TICKET-005 (심화) — 단일 클래스에서 다중 클래스로

**우선순위**: Low (도전 과제)

**배경**
현재 모델은 모든 병변을 `abnormality` 하나로 뭉뚱그립니다. 제품 가치를 높이려면 어떤 종류의 이상인지(14개 소견 분류)까지 구분해야 합니다. 다만 클래스가 늘면 클래스별 데이터 불균형이 드러나고, 희귀 소견의 성능이 낮게 나올 것입니다.

**목표**
14개 다중 클래스로 확장했을 때 클래스별 성능 격차를 측정하고, 어떤 클래스가 문제인지 식별한다.

**Acceptance Criteria**
- [ ] 클래스별 AP 제시
- [ ] AP가 낮은 클래스 식별 및 원인 추정(데이터 개수와 연결)
- [ ] 다음에 무엇을 개선해야 하는지 제안

**Deliverable**: 실험 리포트 1건 (클래스별 AP 표 포함)
